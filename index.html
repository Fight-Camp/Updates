Update · 12 May 2026

FightCamp dashboard rebuild

A round of work on the post-onboarding experience: weight entries now save and persist (online or off), the Home page is a real dashboard, and there's a new Progress tab for the trend view.

Saving · Offline-first

Adding weight now persists

Weight entries used to live in memory only — every reload wiped them. They now save to the database and stick around. We also reworked the input itself to a native iOS wheel picker that pre-fills with the user's most recent weight, so logging today's number is a one- or two-notch adjustment from yesterday.

  • Native iOS wheel: whole-number + tenths columns, unit-aware range (kg / lbs)
  • Saves to Firestore, with offline persistence so writes survive flaky reception
  • When the device reconnects, queued writes sync automatically — no extra UI

Home · Dashboard

Home is now a real dashboard

The Home tab was a placeholder dump of profile fields. It's now a proper dashboard with stat cards, a burnout level indicator, fight-camp context, and historical milestones — all driven by the user's actual data.

  • Daily Weight — hero current weight, week-over-week and camp-total deltas, weight + time remaining, inline quick-add
  • Burnout Level — semicircular gauge that combines planned pace, actual pace, and time pressure
  • Trending Weight Changes — 7-day rolling change with empty-state copy when there's not enough data yet
  • About this fight camp — fight date, target division, target weights
  • Milestones & Weeks — this-week / last-week check-in counts + deltas, plus historical week tiles
  • Pro Corner & Suggested for you — placeholder content slots ready for real photography

Progress · Trend view

New Progress tab

A dedicated screen for the trend story. Three-column summary card up top showing start, current, and total change. Filter pills tell you which series is in focus — the chart itself shows everything: daily entries, the trending line, the plan line, and a dashed forecast projection past today.

  • Start / Current State / Change — same-size numbers, edge-aligned across the card
  • Filter pills: Daily / Trending / Plan, with a blue indicator bar on the active one
  • Chart auto-fits to the data range — no more sprawling y-axis on a tight cut
  • Full weight log below the chart, most recent first

What's behind it

Quiet foundation work

A few less-visible things that make the rest of the app feel solid.

Bottom navigation

Five-tab bar: Home / Plan / + (sheet) / Progress / More — replaces the single-screen post-auth view.

Working Settings

Appearance toggle (Light / Dark / System), Notification preferences with daily-weigh-in time picker, email update, profile view.

Notifications

Local notification scheduling with a daily weigh-in trigger and a deeplink router so taps land in the right place.

Design system

Token-driven radii, spacing, typography, motion, and colors — applied across all 23 onboarding screens.

Pure-value calculators

Burnout score and weight forecasting live in a dependency-free Logic layer that's easy to tweak in one place.

Demo account

Seeded dev user with 39 weight entries and an active fight camp, so the dashboard demos itself.
